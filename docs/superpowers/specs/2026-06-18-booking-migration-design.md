# Spesifikasi Desain: Migrasi Aplikasi Booking Lapangan HAM ke Next.js & Supabase

Dokumen ini mendefinisikan arsitektur sistem, struktur data, alur transaksi, dan desain antarmuka pengguna untuk memigrasi aplikasi **Booking Lapangan HAM** dari *Native PHP & MySQL* ke *Next.js (App Router), TypeScript, Supabase (Auth, Postgres, Storage),* dan *Vercel Deployment*.

---

## 1. Arsitektur Teknologi (Tech Stack)

Sistem baru akan dibangun di atas fondasi berikut:
*   **Frontend & Routing**: Next.js 15+ (App Router) dengan TypeScript.
*   **Desain & UI**: Tailwind CSS v4 untuk styling berkinerja tinggi, dikombinasikan dengan shadcn/ui untuk komponen UI premium berbasis Radix Primitives.
*   **Database**: Supabase PostgreSQL dengan database-level trigger untuk proteksi bentrok jadwal.
*   **Autentikasi**: Supabase Auth terintegrasi dengan tabel profil publik `public.profiles`.
*   **Penyimpanan Berkas (Storage)**: Supabase Storage untuk bukti transfer pembayaran (private) dan gambar lapangan (public).
*   **Notifikasi Email**: Resend API yang dipanggil melalui Next.js Server Actions.
*   **Deployment**: Vercel Platform.

## 2. Skema Database & Relasi Data (PostgreSQL DDL)

Berikut adalah definisi skema database PostgreSQL di Supabase:

### A. Custom Types (Enums)
```sql
CREATE TYPE user_role AS ENUM ('admin', 'customer');
CREATE TYPE booking_status AS ENUM ('pending', 'dp_paid', 'payment_2_pending', 'paid', 'confirmed', 'cancelled');
CREATE TYPE payment_type AS ENUM ('dp', 'final');
```

### B. Tabel `public.profiles`
Tabel profil pengguna yang tersinkronisasi otomatis dengan Supabase Auth.
```sql
CREATE TABLE public.profiles (
    id UUID PRIMARY KEY REFERENCES auth.users ON DELETE CASCADE,
    name TEXT NOT NULL,
    phone TEXT,
    role user_role DEFAULT 'customer' NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

### C. Tabel `public.fields` (Lapangan)
```sql
CREATE TABLE public.fields (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT NOT NULL,
    price NUMERIC(10, 2) NOT NULL,
    address TEXT,
    status TEXT DEFAULT 'active' NOT NULL CHECK (status IN ('active', 'inactive')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

### D. Tabel `public.bookings`
```sql
CREATE TABLE public.bookings (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
    field_id BIGINT NOT NULL REFERENCES public.fields(id) ON DELETE CASCADE,
    booking_date DATE NOT NULL,
    start_time TIME WITHOUT TIME ZONE NOT NULL,
    end_time TIME WITHOUT TIME ZONE NOT NULL,
    price NUMERIC(10, 2) NOT NULL,
    dp_amount NUMERIC(10, 2) NOT NULL,
    status booking_status DEFAULT 'pending' NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

### E. Tabel `public.payments` (Tabel Transaksi Pembayaran)
```sql
CREATE TABLE public.payments (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    booking_id BIGINT NOT NULL REFERENCES public.bookings(id) ON DELETE CASCADE,
    amount NUMERIC(10, 2) NOT NULL,
    payment_type payment_type NOT NULL,
    receipt_url TEXT NOT NULL,
    status TEXT DEFAULT 'pending' NOT NULL CHECK (status IN ('pending', 'approved', 'rejected')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

## 3. Otomatisasi & Logika Database (Triggers & Functions)

### A. Sinkronisasi User Baru (`handle_new_user`)
Mengisi tabel `public.profiles` secara otomatis ketika user sukses mendaftar di Supabase Auth.
```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, name, phone, role)
  VALUES (
    NEW.id,
    COALESCE(NEW.raw_user_meta_data->>'name', 'User'),
    NEW.raw_user_meta_data->>'phone',
    'customer'
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

### B. Proteksi Bentrok Jadwal (`prevent_double_booking`)
Mencegah tumpang tindih waktu sewa lapangan di tingkat database.
```sql
CREATE OR REPLACE FUNCTION public.prevent_double_booking()
RETURNS TRIGGER AS $$
DECLARE
    conflict_count INT;
BEGIN
    SELECT COUNT(*) INTO conflict_count
    FROM public.bookings
    WHERE field_id = NEW.field_id
      AND booking_date = NEW.booking_date
      AND status != 'cancelled'
      AND id != COALESCE(NEW.id, -1)
      AND (
        (NEW.start_time, NEW.end_time) OVERLAPS (start_time, end_time)
      );

    IF conflict_count > 0 THEN
        RAISE EXCEPTION 'DoubleBookingException: Slot waktu pada tanggal ini telah dibooking oleh pelanggan lain.';
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE OR REPLACE TRIGGER before_booking_upsert
  BEFORE INSERT OR UPDATE ON public.bookings
  FOR EACH ROW EXECUTE FUNCTION public.prevent_double_booking();
```

## 5. Keamanan Penyimpanan Berkas (Supabase Storage RLS)

### A. Bucket `field-images` (Public)
*   **Izin Read**: Publik (`true`).
*   **Izin Write / Update / Delete**: Hanya admin.
    ```sql
    -- Kebijakan contoh untuk Write:
    (select role from public.profiles where id = auth.uid()) = 'admin'
    ```

### B. Bucket `payment-proofs` (Private)
*   **Izin Read**: Hanya user pengunggah dan admin.
    ```sql
    auth.uid()::text = (storage.foldername(name))[1] OR 
    (select role from public.profiles where id = auth.uid()) = 'admin'
    ```
*   **Izin Write**: Hanya authenticated user.
    ```sql
    auth.role() = 'authenticated'
    ```

## 6. Logika Perhitungan Tarif Slot (`src/config/pricing.ts`)

Aturan tarif slot statis akan diimplementasikan sebagai berkas konfigurasi TypeScript bersama:
```typescript
export interface PriceSlot {
  startHour: number;
  endHour: number;
  weekdayPrice: number | null;
  fridayPrice: number | null;
  weekendPrice: number;
}

export const BOOKING_PRICE_SLOTS: PriceSlot[] = [
  { startHour: 6, endHour: 8, weekdayPrice: null, fridayPrice: null, weekendPrice: 2600000 },
  { startHour: 8, endHour: 10, weekdayPrice: 1300000, fridayPrice: null, weekendPrice: 2600000 },
  { startHour: 10, endHour: 12, weekdayPrice: 800000, fridayPrice: null, weekendPrice: 1500000 },
  { startHour: 12, endHour: 14, weekdayPrice: 800000, fridayPrice: null, weekendPrice: 1500000 },
  { startHour: 14, endHour: 16, weekdayPrice: 1300000, fridayPrice: 1300000, weekendPrice: 2600000 },
  { startHour: 16, endHour: 18, weekdayPrice: 2000000, fridayPrice: 2300000, weekendPrice: 2600000 },
  { startHour: 18, endHour: 20, weekdayPrice: 2300000, fridayPrice: 2500000, weekendPrice: 2800000 },
  { startHour: 20, endHour: 22, weekdayPrice: 2300000, fridayPrice: 2500000, weekendPrice: 2800000 }
];

export function calculateBookingPrice(date: Date, startHour: number, endHour: number): { total: number; dp: number } {
  const day = date.getDay(); // 0 = Minggu, 5 = Jumat, 6 = Sabtu
  let total = 0;

  // Evaluasi setiap slot yang terlewati
  for (const slot of BOOKING_PRICE_SLOTS) {
    if (slot.startHour >= startHour && slot.endHour <= endHour) {
      let price = slot.weekendPrice;
      if (day >= 1 && day <= 4) { // Senin - Kamis
        price = slot.weekdayPrice ?? slot.weekendPrice;
      } else if (day === 5) { // Jumat
        price = slot.fridayPrice ?? slot.weekendPrice;
      }
      total += price;
    }
  }

  // DP minimal 30% dari total biaya
  const dp = Math.max(total * 0.3, 500000); 

  return { total, dp };
}
```

## 7. Rencana Pengujian (Verification Plan)

### A. Pengujian Otomatis
*   **Unit Testing (Vitest)**: Memastikan fungsi `calculateBookingPrice` mengembalikan angka total dan nominal DP yang akurat untuk berbagai kombinasi hari dan rentang jam.
*   **Database Constraints Test**: Menulis skrip uji migrasi sql untuk memvalidasi bahwa database menolak insert pemesanan ganda di slot waktu yang bertubrukan.

### B. Pengujian Manual
*   **Alur Pengguna (E2E)**: Mendaftar akun baru $\rightarrow$ booking slot $\rightarrow$ unggah bukti transfer DP $\rightarrow$ verifikasi admin $\rightarrow$ unggah bukti pelunasan $\rightarrow$ konfirmasi final oleh admin.
*   **Pengujian Ephemeral File System**: Mengonfirmasi berkas bukti transfer tidak hilang setelah fungsi serverless di-recycle/restart.
